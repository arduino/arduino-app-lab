### Conventions

#### Method conventions
Api request methods exposed from the infrastructure layer are always prefixed with "Request", to give more naming liberty in the service layer.
Link to related [discussion](https://github.com/bcmi-labs/cloud-editor-mono/pull/89#issuecomment-1513281516).

### Type conventions for builderApi and createApi

"_Params" represents url route or query string params for an endpoint
"_Body" represents the body for an endpoint
"_BuilderApi" a type from related api docs
"_Response" represents mapped data from an endpoint
